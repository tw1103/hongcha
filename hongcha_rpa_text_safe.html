推荐定位优先级

元素定位建议
优先级 属性 示例 说明
1 id login-submit-button 页面内唯一，稳定性最高
2 data-rpa-id transaction-query-button 专门为自动化定位设计
3 name transactionDate 适合表单字段定位
4 文本 + 父级区域 流水数据查询 / transaction-page 用于无法直接按属性定位的场景
下拉框捕获： 页面中的可见下拉框采用 button + ul + li + div 结构，触发器为 button，每个选项都是普通 div DOM。先点击下拉框触发按钮，再捕获具体选项。例如：“类型A”的固定 ID 为 transaction-type-select-option-002。
本页面未使用 iframe、Canvas、Shadow DOM、虚拟列表或随机 ID。所有业务页面在初始加载时已经存在于 DOM 中，仅通过 hidden 属性切换显示；表格数据使用固定行键。原生 select 仅作为隐藏值容器，可见下拉选项均为可捕获的普通按钮元素。
